## What is tweens?

tweens uses spring physics instead of fixed easing curves. You describe a spring — stiffness, damping, bounce — and the element settles to its target on its own schedule.

Springs are **interruptible**: start a new animation mid-flight and the element picks up from wherever it is, carrying its velocity into the new direction. No snap, no timer to cancel.

tweens also resolves a **Promise** when the spring settles, so you can sequence animations with `await` instead of nesting callbacks.

## Why not CSS transitions?

CSS transitions have a fixed duration. If you interrupt one mid-way, the element snaps back to the start of the new transition. Springs don't — they carry velocity through any interruption, which is what makes hover states and drag gestures feel fluid.

## Why not GSAP or Motion?

GSAP is powerful but ~40 KB gzipped. Motion's vanilla package is lighter but still heavier than needed for most use cases. tweens is **~2 KB gzipped**, has zero dependencies, and works on any DOM element — React, Vue, Svelte, or plain HTML.

## Install

```bash
npm install @tweens/tweens
```

## Quick example

```ts
import { spring } from '@tweens/tweens'

const card = document.getElementById('card')

// Slide right
spring(card, { x: 100 }, 'snappy')

// Fade in — returns a Promise
await spring(card, { opacity: 1 }, 'gentle')

// Chain animations
await spring(card, { y: -8 })
await spring(card, { scale: 1.05 })
```

All animation functions return `Promise<void>` that resolves when the spring physically settles.

## Browser support

Any browser with `requestAnimationFrame` and `WeakMap` — all modern browsers. Automatically respects `prefers-reduced-motion`.
