---
title: cascade() / sequence()
description: Orchestration helpers. cascade() staggers a list with a time offset. sequence() runs animations one after another.
---

## cascade()

Animates a list of elements with a delay between each start. All animations run concurrently — they just begin at staggered times.

<CascadeDemo />

```ts
import { cascade } from '@tweens/tweens'

const items = document.querySelectorAll('.item')

// Stagger in — 70ms between each
cascade(items, { y: 0, opacity: 1 }, 'snappy', 0.07)

// In React
cascade(itemRefs.current, { y: 0, opacity: 1 }, 'gentle', 0.05)
```

Before calling `cascade()`, reset the elements to their starting state with `snap()`:

```ts
els.forEach(el => snap(el, { y: 16, opacity: 0 }))
cascade(els, { y: 0, opacity: 1 }, 'snappy', 0.07)
```

### Signature

```ts
cascade(
  els: NodeListOf<Element> | HTMLElement[],
  props: AnimateProps,
  options?: AnimateOptions | SpringPreset,
  interval?: number,    // seconds between each, default 0.05
): Promise<void>
```

## sequence()

Runs each step serially — the next animation starts only after the previous one settles.

<SequenceDemo />

Each square waits for the one before it to finish. The difference from `cascade()`: cascade staggers but runs in parallel, sequence is strictly serial.

```ts
import { sequence, spring, enter, cascade } from '@tweens/tweens'

await sequence([
  () => enter(header, { y: 20, opacity: 0 }, 'snappy'),
  () => enter(body, { y: 16, opacity: 0 }, 'gentle'),
  () => cascade(cards, { y: 0, opacity: 1 }, 'snappy', 0.06),
])
```

Because `sequence()` accepts functions that return Promises, you can mix any tweens calls — or any Promise-returning function.

### Signature

```ts
sequence(
  steps: Array<() => Promise<void>>,
): Promise<void>
```
