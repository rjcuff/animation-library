Pass a preset string as the third argument to any spring function instead of a config object.

```ts
spring(el, { scale: 1.04 }, 'snappy')
enter(el, { opacity: 0, y: 12 }, 'gentle')
animate(el, { x: 100 }, 'bouncy')
```

## When to use each

**`bouncy`** — Elements that should feel playful or alive. Draggable items snapping back, icon button feedback, anything you want to feel lively.

**`snappy`** — The default for most hover and interaction states. Fast enough to feel responsive, controlled enough to feel professional.

**`gentle`** — Page entrances, modals fading in, content reveals. Slow enough to feel intentional without being sluggish.

**`stiff`** — Press feedback (scale down on `mousedown`), immediate UI state responses where any bounce would feel laggy or wrong.

## Custom physics

When the presets don't fit, configure the spring directly with `stiffness`, `damping`, and `mass`.

```ts
spring(el, { x: 100 }, {
  stiffness: 300,  // how rigid the spring is
  damping:   20,   // how quickly oscillation decays
  mass:      1,    // heavier = slower to start and stop
})
```

The damping ratio `ζ = damping / (2 × √(stiffness × mass))` determines behavior:
- `ζ < 1` — overshoots and oscillates (underdamped)
- `ζ = 1` — fastest settle with zero overshoot (critically damped)
- `ζ > 1` — slow creep to target (overdamped)

## Feel-based config

If you don't want to think in physics units, use `duration` + `bounce`:

```ts
spring(el, { x: 100 }, {
  duration: 0.4,  // seconds until settled
  bounce:   0.2,  // 0 = no overshoot, 1 = maximum oscillation
})
```
