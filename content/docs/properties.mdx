## AnimateProps

All properties are optional. Mix freely — multiple props animate in parallel.

```ts
animate(el, {
  x: 100,
  y: -20,
  scale: 1.05,
  opacity: 0.8,
}, 'snappy')
```

| Prop | Unit | CSS output | Default |
|------|------|------------|---------|
| `x` | px | `translateX()` | 0 |
| `y` | px | `translateY()` | 0 |
| `scale` | — | `scale()` | 1 |
| `rotate` | deg | `rotate()` | 0 |
| `opacity` | 0–1 | `opacity` | 1 |
| `blur` | px | `filter: blur()` | 0 |
| `width` | px | `width` | computed |
| `height` | px | `height` | computed |
| `borderRadius` | px | `border-radius` | computed |
| `backgroundColor` | hex/rgb/rgba | `background-color` | computed |
| `color` | hex/rgb/rgba | `color` | computed |

`x`, `y`, `scale`, and `rotate` compose correctly — animating `x` won't disrupt a separately running `scale` animation.

## Color format

`backgroundColor` and `color` accept:

```ts
'#3b82f6'               // hex 6-digit
'#f00'                  // hex 3-digit shorthand
'rgb(59, 130, 246)'     // rgb()
'rgba(59, 130, 246, 1)' // rgba() — alpha ignored, RGB preserved
```

## Performance

Only `transform` (x, y, scale, rotate) and `opacity` are GPU-composited — these skip layout and paint entirely. `blur` uses `filter`, which is composited on most devices. `width`, `height`, `borderRadius`, and color properties trigger layout or paint; use them for structural transitions, not continuous high-frequency animations.

`will-change` is set automatically on animation start and cleared when all animations on an element settle.

## SpringConfig

```ts
interface SpringConfig {
  // Physics params
  stiffness?: number        // spring constant, default 170
  damping?: number          // friction, default 26
  mass?: number             // inertia, default 1

  // Feel-based (alternative to physics)
  duration?: number         // seconds until settled
  bounce?: number           // 0 = no overshoot, 1 = max oscillation

  // Shared
  delay?: number            // seconds before start
  repeat?: number           // extra plays; -1 = infinite
  yoyo?: boolean            // reverse direction each repeat
  onStart?: () => void
  onUpdate?: (v: number) => void
  onComplete?: () => void
}
```

## EasingConfig

```ts
interface EasingConfig {
  duration: number
  ease?: EasingName | EasingFn | [x1: number, y1: number, x2: number, y2: number]

  delay?: number
  repeat?: number
  yoyo?: boolean
  onStart?: () => void
  onUpdate?: (v: number) => void
  onComplete?: () => void
}

type EasingName =
  // ease-out family (enter/exit animations)
  | 'easeOutQuad' | 'easeOutCubic' | 'easeOutQuart'
  | 'easeOutQuint' | 'easeOutExpo' | 'easeOutCirc'
  // ease-in-out family (on-screen movement)
  | 'easeInOutQuad' | 'easeInOutCubic' | 'easeInOutQuart'
  | 'easeInOutQuint' | 'easeInOutExpo' | 'easeInOutCirc'
  // hover & color transitions
  | 'ease'
  // constant motion
  | 'linear'
  // CSS equivalents
  | 'easeIn' | 'easeOut' | 'easeInOut'

type EasingFn = (t: number) => number  // t ∈ [0, 1]
```

Default ease is `'easeOutCubic'` when none is specified.

## snap() and stop()

```ts
import { snap, stop } from '@tweens/tweens'

// Set properties instantly — cancels any running animation for those props
snap(el, { x: 0, y: 0, scale: 1, opacity: 1 })

// Cancel all animations on an element
stop(el)

// Cancel a specific property
stop(el, 'x')
```
