Pass a preset string as the third argument to any spring function instead of a config object.

```ts
spring(el, { scale: 1.04 }, 'snappy')
enter(el, { opacity: 0, y: 12 }, 'gentle')
animate(el, { x: 100 }, 'bouncy')
```

## When to use each

**`bouncy`** is for things that should feel lively. Draggable items snapping back, icon button feedback, anything you want to feel playful and alive.

**`snappy`** is the default for most hover and interaction states. Fast and decisive without feeling wild.

**`gentle`** suits page entrances, modals opening, and content reveals. Slow enough to feel intentional.

**`stiff`** is for press feedback. No overshoot, no bounce — just an instant compress and release.

## Custom physics

When the presets don't fit, configure the spring directly with `stiffness`, `damping`, and `mass`.

```ts
spring(el, { x: 100 }, {
  stiffness: 300,  // how rigid the spring is
  damping:   20,   // how quickly oscillation decays
  mass:      1,    // heavier = slower to start and stop
})
```

Higher stiffness makes the spring faster and snappier. Higher damping reduces bounce. Lower damping with high stiffness gives you overshoot and oscillation. Mass makes the element feel heavier — slower to get moving and slower to stop.

## Feel-based config

If you don't want to think in physics units, use `duration` + `bounce`:

```ts
spring(el, { x: 100 }, {
  duration: 0.4,  // seconds until settled
  bounce:   0.2,  // 0 = no overshoot, 1 = maximum oscillation
})
```
