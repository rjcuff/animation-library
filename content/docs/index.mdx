## What is tweens?

tweens is a lightweight animation library that makes DOM elements move with **spring physics** — the same model used in iOS, Android, and Motion. Instead of picking an easing curve and a duration, you describe a spring: how stiff it is, how much it bounces. The element then physically settles to its target.

The result is animation that feels responsive and alive rather than mechanical. Springs are also **interruptible** — if you start a new animation mid-way, the element continues from its current position and velocity with no snapping or jarring.

## Why not CSS transitions?

CSS transitions have a fixed duration and easing curve. If you interrupt one mid-way (hover off before it finishes), the element snaps to a new starting point. Springs don't — they carry velocity through any interruption, which is what makes interactions feel fluid.

tweens also gives you a **Promise** when the animation settles, so you can sequence animations with `await` without managing timers.

## Why not GSAP or Motion?

GSAP is powerful but large (~40 KB gzipped). Motion requires React and adds significant bundle weight for a relatively simple task. tweens is **~2 KB gzipped**, has zero dependencies, and works on any DOM element — React, Vue, Svelte, or plain HTML.

## Install

```bash
npm install @tweens/tweens
```

## Quick example

```ts
import { spring } from '@tweens/tweens'

const card = document.getElementById('card')

// Slide right with a bouncy spring
spring(card, { x: 100 }, 'bouncy')

// Fade in — returns a Promise
await spring(card, { opacity: 1 }, 'gentle')

// Chain animations
await spring(card, { y: -8 })
await spring(card, { scale: 1.05 })
```

All animation functions return `Promise<void>`. The promise resolves when the spring physically settles — no guessing at durations.

## Browser support

Any browser that supports `requestAnimationFrame` and `WeakMap` — effectively all modern browsers. Automatically respects `prefers-reduced-motion`.
