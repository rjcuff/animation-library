---
title: cascade() / sequence()
description: Orchestration helpers. cascade() staggers a list. sequence() runs animations one after another.
---

## cascade()

Animates a list of elements with a delay between each start. All animations run concurrently — they just begin at staggered times.

<CascadeDemo />

```ts
import { cascade } from '@tweens/tweens'

const items = document.querySelectorAll('.item')

// Stagger in — 60ms between each
cascade(items, { y: 0, opacity: 1 }, 'snappy', 0.06)
```

Reset elements to their starting state before calling `cascade()`:

```ts
els.forEach(el => snap(el, { y: 12, opacity: 0, scale: 0.92 }))
cascade(els, { y: 0, opacity: 1, scale: 1 }, 'snappy', 0.06)
```

### Signature

```ts
cascade(
  els: NodeListOf<Element> | HTMLElement[],
  props: AnimateProps,
  options?: AnimateOptions | SpringPreset,
  interval?: number,    // seconds between each start, default 0.05
): Promise<void>
```

## sequence()

Runs each step serially — the next animation starts only after the previous settles.

<SequenceDemo />

Unlike `cascade()` (staggered but parallel), `sequence()` is strictly serial.

```ts
import { sequence, spring, enter, cascade } from '@tweens/tweens'

await sequence([
  () => enter(header, { y: 20, opacity: 0 }, 'snappy'),
  () => enter(body, { y: 16, opacity: 0 }, 'gentle'),
  () => cascade(cards, { y: 0, opacity: 1 }, 'snappy', 0.06),
])
```

`sequence()` accepts any function that returns a Promise, so you can mix tweens calls with any async operation.

### Signature

```ts
sequence(
  steps: Array<() => Promise<void>>,
): Promise<void>
```
