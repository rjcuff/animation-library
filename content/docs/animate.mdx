---
title: animate()
description: The unified engine. Handles spring physics and easing curves — one function, two modes.
---

`animate()` is what `spring()`, `enter()`, and `tween()` all call internally. You can use it directly when you need easing curves instead of (or alongside) springs.

The mode is determined by the `ease` key:

```ts
import { animate } from '@tweens/tweens'

// Spring mode — physics, no fixed duration
animate(el, { x: 100 }, 'bouncy')
animate(el, { x: 100 }, { stiffness: 300, damping: 20 })

// Easing mode — 'ease' key switches the mode
animate(el, { x: 100 }, { duration: 0.4, ease: 'easeOut' })
animate(el, { x: 100 }, { duration: 0.5, ease: 'easeOutBack' })
```

## Spring vs easing

Both use the same API. The difference is in feel — springs overshoot and settle naturally, easings follow a fixed curve.

<AnimateDemo />

Bouncy spring overshoots the target; easeOutBack touches the target then pulls back. Both start and end in the same place.

## Color animation

Animate `backgroundColor` and `color` between any hex or `rgb()` value. Each click cycles through colors.

<ColorDemo />

Color animation uses easing mode. Colors always stay valid — no clamping needed on your end.

```ts
animate(el, { backgroundColor: '#3b82f6' }, { duration: 0.25, ease: 'easeOut' })
animate(el, { color: 'rgb(255, 255, 255)' }, { duration: 0.3, ease: 'easeOut' })
```

## Easing names

```ts
'linear'
'easeIn'   | 'easeOut'   | 'easeInOut'
'easeInBack' | 'easeOutBack'
'easeInElastic' | 'easeOutElastic'

// Or a custom function:
animate(el, { x: 100 }, { duration: 0.4, ease: (t) => t * t })
```

## Signature

```ts
animate(
  el: HTMLElement,
  props: AnimateProps,
  options?: AnimateOptions | SpringPreset,
): Promise<void>
```

## Shared options

Both spring and easing modes support these:

```ts
animate(el, { x: 100 }, {
  duration: 0.4,
  ease: 'easeOut',
  delay:      0.1,   // seconds before start
  repeat:     2,     // extra plays after first; -1 = infinite
  yoyo:       true,  // reverse on every other repeat
  onStart:    () => {},
  onUpdate:   (v) => {},
  onComplete: () => {},
})
```
