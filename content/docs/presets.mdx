title: Presets
description: Four named spring configs. Pass as a string instead of a config object.

Pass a preset string as the third argument to any spring function:

```ts
spring(el, { scale: 1.03 }, 'snappy')
enter(el, { opacity: 0, y: 12 }, 'gentle')
animate(el, { x: 100 }, 'bouncy')
```

## When to use each

**`snappy`** is the default for most interactions — hover, focus, card lifts. Fast and settled with no bounce.

**`gentle`** for entrances: modals, drawers, page sections. Unhurried but not slow.

**`bouncy`** has a subtle overshoot. Good for drag-to-dismiss, icon feedback, or any interaction that should feel elastic. Not the right choice for most UI — use `snappy` as your baseline.

**`stiff`** for press feedback. Instant compress — users feel direct contact.

## Custom physics

When presets don't fit, configure directly:

```ts
spring(el, { x: 100 }, {
  stiffness: 300,  // spring constant
  damping:   28,   // friction
  mass:      1,    // inertia
})
```

Stiffness → speed. Damping → friction (higher = less bounce). Mass → inertia (higher = slower start and stop).

## Feel-based config

Skip the physics parameters with `duration` + `bounce`:

```ts
spring(el, { x: 100 }, {
  duration: 0.4,  // seconds until settled
  bounce:   0.15, // 0 = no overshoot, 1 = maximum oscillation
})
```

Keep `bounce` between 0 and 0.3 for most UI. Higher values get theatrical fast.
